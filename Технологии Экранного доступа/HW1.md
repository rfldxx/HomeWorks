## Домашнее задание № 1
Создать текстовый документ, поместить туда произвольный текст и сохранить документ.

### Ход работы:
1. Первое включение NVDA:
    - NVDA запускается комбинцией клавиш: `ctrl + alt + N`
    - при запуске откроется небольшое окно приветствия (при желании в этом окне можно отключить флажок: "Показывать этот диалог при запуске NVDA")
    - первоначально фокус находится на поле "раскладка клавиатуры": стрелкой вверх / вниз можно переключать `настольная` / `ноутбук` (между ними отличаются комбинации клавиш)
    - перейти к следующему полю можно через нажатие клавиши: `tab`
    - NVDA в своих комбинациях использует выделенную клавишу-модификатор `nvda`, которая по умолчанию назначена на клавишу `insert`
    - в одном из следующих полей можно включить флаг: "Использовать CapsLock как клавишу-модификатор NVDA"
    - флаг включается нажатием на `пробел` либо на `Enter`

2. Открытие приложения:
    - открыть поиск в Windows: `win`
    - начнём вводить название приложения "блокнот" 
    - после каждой нажатой клавиши скринридер будет озвучивать первое предлагаемое приложение
    - предлагаемое приложение автоматически попадает в фокус и можно:
        - нажать `Enter` чтобы запустить приложение
        - нажать `стрелка вправо` чтобы перейти к дополнительным действиям (там навигируемся стрелками вверх / вниз, либо табом)
        - у некоторых приложений часть дополнительных действий свёрнута: надо нажать на `пробел`/`Enter`, чтобы раскрыть список 

3. Работа с текстом:
    - при навигации стрелками NVDA автоматически читает текст:
        - при стрелка вверх / вниз: озвучивается вся строчка на которую перешел курсор
        - при стрелка ввлево / вправо: озвучивается символ **справа** от положения куда перешёл курсор 
        - при зажатом `ctrl` + стрелка ввлево / вправо (переход по словам): озвучивается слово **справа** от положения куда перешёл курсор
    
    - также можно читать текст не перемещая курсора (см. "Режим обзора"):
        - для этого надо зажать клавишу `nvda` + любой вариант из предыдушего пункта
        - можно воспринимать будто создался "виртуальный курсор" на текущей позиции и мы его перемещаем комбинациями: `nvda + стрелочка`
        - при этом "виртуальный курсор" перемещается и сохраняет новое положение (и может очень сильно отдрейфовать от начального положения)
        - если мы двинем "обычный курсор", то "виртуальный курсор" сразу вернется обратно к "обычному курсору" (или можно сказать удалиться)
    
    - вообще больше комбинаций для работы с текстом можно прочитать в разделе: [Навигация с NVDA](https://download.nvaccess.org/releases/stable/documentation/ru/keyCommands.html#NavigatingWithNVDA)

4. Итого, я напечал текст в блокноте, нажал `ctrl + S` чтобы сохранить файл:
    - меня перекинуло в "сохранение-диалог" (как сказал NVDA, хотя по сути открылся проводник)
    - фокус находится на поле: "Имя файла" сразу переименовал файл (хотя можно будет табами/шифт-табами обратно вернуться)
    - и теперь остается только указать "адресс" сохранения

5. Навигация в проводнике:
    - первым делом перейдём к дереву: при открытие проводника из стороннего приложения, до него проще всего добраться через `shift + tab`
    - навигация по дереву осуществляется с помощбю стрелок:
        - `стрелка вправо`: развернуть список
        - `стрелка влево`: свернуть список, если список уже свёрнут, то перейти к предку текущего объекта
        - `стрелка вверх` / `стрелка вниз` итрерирование по списку
    - как нашли нужную папку в дереве нажимаем `Enter` (по умолчанию NVDA ничего дополнительного не говорит)
    - если хотим узнать точное местоположение, можем ещё несколько раз понажимать `shift + tab` пока не озвучится поле "адресс", потом с небольшой задержкой NVDA начнет читать адресс
    - если в поле "адресс" нажать `Enter` то это поле становится доступным для редактирования и при этом весь адресс выделится

6. Наконец локализовавшись, жмём `tab` пока не дойдем до: "сохранить копка". \
Однако на этой кнопки NVDA озвучил `Alt + х` (х - русская буква) - и действительно такая комбинация тоже работает.
